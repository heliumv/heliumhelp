Intelligenter Stücklistenimport

Mit diesem zusätzlichen Modul steht ein strukturierter Import anhand verschiedenen Textformate in die Stücklisten zur Verfügung.

 

Hinweis:

Diese Funktionalität benötigt erweiterte SQL Funktionen und steht derzeit nur unter PostgreSql zur Verfügung.

 

Nach dem Anlegen der Stücklisten Kopfdaten wechseln Sie in den oberen Modulreiter Positionen.

Um die Importfunktion zu starten klicken Sie auf den Knopf Import.

Im nun erscheinenden Dialog

wählen Sie eine Textdatei mit den Stücklistendaten durch Klick auf aus.

 

Nun erhalten Sie eine entsprechende Vorschau der Textdaten.

Definieren Sie nun anhand der Vorschau die Positionen der Felder

Bauteil Die Bezeichnung des Artikels bzw. dessen Artikelnummer in Ihrem Helium V Artikelstamm
Die dafür im Artikelstamm durchsuchten / zugeordneten Felder sind:
Bauform Hier wird in Helium V das Feld Bauform (Reiter Technik)  durchsucht
Menge Der numerische Wert dieser Spalte wird als Menge übernommen
Positionsnummer Der Inhalt dieser Spalte wird unter Positionsnummer eingetragen.

 

Unter dem Dateiauswahldialog definieren Sie

 

Anzahl der Kopfzeilen  Nach der Anzahl dieser Zeilen beginnen die echten Daten
Trennzeichen
Kopfzeile mit Feldnamen D.h. aus der Kopfzeile können die Feldnamen herausgelesen werden. Die erwarteten Feldnamen sind:

 

Definieren Sie nun die Positionen der Felder im unteren Teil des Dialoges.

D.h. markieren Sie z.B. das Feld Positionsnummer mit der Maus und klicken Sie dann
auf die beiden doppelten Pfeile.
Damit werden Beginn und Ende der Markierung als die Position in der Textdatei für das gewählte Feld, in unserem Falle die Positionsnummer, übernommen.

 

Nun klicken Sie auf Vorschau um im unteren Teil die aufgesplitteten Daten zu sehen.

Sind diese so wie von Ihnen gewünscht, so klicken Sie bitte auf Import

Nun wird versucht anhand der angegebenen Daten aus Bauteil und Bauform den je Zeile am besten passenden Artikel in Ihrem Artikelstamm zu finden.

Nach einigen Sekunden, abhängig von der Menge der Daten erhalten Sie die Import-Liste in der Sie die Übersetzungen zu Ihren Artikeln sehen. Beachten Sie dabei bitte die Spalte gefunden. Hier wird angezeigt wieviele passenden Artikel zu diesem Text gefunden wurden. Wurden mehr wie 1 Artikel gefunden, so kann durch Klick in das Feld Artikel aus den weiteren gefundenen Artikeln ausgewählt werden.

D.h. prüfen Sie die Liste und passen Sie die Ergebnisse so an, dass der richtige Artikel (aus Ihrem Artikelstamm) bei der jeweiligen Position eingetragen ist.

Da oft viele gleichlautende Positionen  (=Zeilen) gegeben sind, gibt es links unten den Knopf Übernehmen. Damit wird die von ihnen getroffene Artikeldefinition in die gleichlautenden Positionen übernommen. Gleichlautend bedeutet, dass, ausgehend von der selektierten Zeile alle Zeilen aktualisiert werden, bei denen Bauteil und Bauform den Werten der gewählten Zeile entsprechen.

Hier können einfach auch Artikelnummern in der Artikelnummernspalte eingegeben werden. Öffnen Sie dazu gegebenenfalls einen zweiten Helium V Client um den richtigen Artikel zu finden.

 

Sind nun alle Zeilen richtig eingestellt, so wird durch Klick auf die Übertragung in die Stücklistenpositionen gestartet. Es werden dabei alle Positionen für die anhand der Artikelnummer (der Import-Liste) ein Artikel gefunden wurde als Ident-Nummern Position übernommen. Alle anderen Positionen (bei denen das gefunden auf 0 steht) werden so übernommen, dass wenn unter Bezeichnung (rechte Spalte) etwas eingegeben wurde, wird diese Eingabe in die Bezeichnung der Handeingabe übernommen. Ansonsten wird der Inhalt von Bauteil und Bauform in die Bezeichnung der Handeingabe eingetragen.

 

Nun wird das Fenster geschlossen und Sie sehen die übernommenen Daten in Ihrer Stückliste.

 

Q: Wie wird gesucht?

A: Es werden im Artikel die Felder Bezeichnung und Artikelnummer nach den Feldern aus Bauteil und Bauform durchsucht.

Die Texte in Bezeichnung haben eine höhere Wertigkeit.

Die zutreffenden Werte die in der Bezeichnung weiter links stehen haben eine höhere Wertigkeit.

Das bedeutet: Damit die Suche gute Daten findet, ist es wichtig dass:

- die Bezeichnungen in Ihrem Artikelstamm immer in der gleichen Art und Weise bezeichnet werden

- dass die wichtigen Daten (z.B. 10nF) weiter links stehen als nicht so wichtige Daten (z.B. Toleranz).
