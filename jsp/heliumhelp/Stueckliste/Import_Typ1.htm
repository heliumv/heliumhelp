Stücklistenimport Typ1 
Solid Works XLS

Mit diesem  Import steht ein Strukturierter Import anhand der von Solid Works erzeugten Stücklistenstruktur, welche normalerweise im XLS Format ausgegeben wird, zur Verfügung. Sie können damit komplette Maschinen in einem Zug nach Helium V übernehmen.

 

Der besondere Vorteil dieses Imports ist, dass Artikel die im Import definiert sind, aber noch nicht im Artikelstamm von Helium V angelegt sind, neu angelegt werden. Zugleich werden gegebenenfalls dafür vorhandene Zeichnungen (PDF-Dateien) automatisch dem Artikel zugeordnet.

Ebenso wird die komplette Stücklistenstruktur entsprechend aufgebaut.

 

Zusätzlich steht hier auch noch der Import in den Anfragevorschlag zur Verfügung.

D.h. Sie können, neben der Fertigungssteuerung für den tatsächlichen bedarf auch sehr komfortabel die anzufragenden Daten in den Anfrage Vorschlag übernehmen. Da in der Importdatei auch die Liefergruppe jedes Artikels mit übergeben wird, kann so auf einfache Weise eine komplette Liefergruppenanfrage an Ihre Lieferanten mit entsprechenden Zusatzinformatinoen der Zeichnungen gestellt werden.

 

Aus den Stücklisten wird der übliche Helium V Weg beschritten. D.h. es werden aufgrund der . Der Trick liegt dann darin, dass aus den Anfragen die tatsächlichen Lieferanten in den Artikelstamm eingepflegt werden und somit dann die Bestellungen automatisch gemacht werden können.

 

 

Für den Import nach Typ1 Solid Works muss der Mandantenparameter STRUKTURIERTER_STKLIMPORT auf 1 gestellt werden.

Nach dem Neustart des Moduls Stückliste finden Sie unter 

den Punkt Import.

 

 

Die Stücklisten werden strukturiert importiert.
Abhängig von der Pos.Nr werden die jeweiligen Positionen in eine Stückliste übernommen.
Alle Artikel werden angelegt bzw. geprüft ob die Bezeichnungen mit dieser Identnummer exakt übereinstimmen.
Ist die Übereinstimmung nicht exakt gegeben, so wird der alte Artikel auf _001 umbenannt und versteckt und der Artikel neu angelegt, mit den Daten aus dem Import File.
Zugleich werden die Zeichnungen aus der Filestruktur importiert. D.h. gibt es im Pfad der Importdatei eine Datei die den gleichen Filenamen wie die Artikelnummer hat, mit beliebiger Endung, so wird die erste gefundene Datei als Kommentar für diesen Artikel übernommen. Wo genau dieser Artikel ausgedruckt werden sollte kann unter ... definiert werden.

Um für die Stücklisten auch entsprechende Arbeitspläne effizient erzeugen zu können, sollten Sie in den mandantenparametern auch den Default Arbeitsplan definiert haben.

Die Abmessungen werden in der Zusatzbezeichnung2 des Artikels hinterlegt
Das Material in der Bauform, der Typ in der Liefergruppe
Ein Artikel gilt dann als derselbe Artikel, wenn Ident/Bezeichnung/Abmessungen und Material gleich sind.

Wenn im Importverzeichnis mehrere Dateien vorhanden sind, die wie die Artikelnummer lauten, dann wird immer die erste Datei als Anhang hinterlegt.

Der Trenner für die Ebenendefinition ist der Punkt "."
