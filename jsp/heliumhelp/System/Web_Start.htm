Web-Start

 

Der Web-Start von HELIUM V bietet eine komfortable Möglichkeit für den Aufruf des Clients.  Mit einem Klick können Sie über das Internet die Java-Anwendung starten. Der Vorteil des Webstarts ist, dass die Aktualisierung des Clients nicht mehr lokal auf Ihrem Rechner eingerichtet werden muss, sondern dies zentral auf Ihrem HELIUM V Server erfolgt. Es wird eine Kopie des HELIUM V Clientprogramms lokal auf Ihrem Rechner abgelegt, welche bei jedem Start (des Links) geprüft wird und gegebenenfalls wird die Clientinstallation aktualisiert.

Das bedeutet, dass der Installations und Update Ablauf für Sie als Anwender stark vereinfacht wird. Mit dem Web-Start verhindern Sie zum Beispiel das Speichern in einem anderen Ordner und sparen bei Updates Zeit.

Um die Sicherheit zu gewährleisten, wurden und werden selbstverständlich von HELIUM V Schritte durchgeführt um eine Manipulation zu verhindern. Ebenso entsprechen die Rechte, die der Web-Start Client an Ihrem Rechner hat denen der lokal gespeicherten HELIUM V Client-Anwendung.

 

Vorraussetzung zur Nutzung des HELIUM V Web-Starts:

  • HELIUM V stellt die Anwendung eine spezielle XML-Datei (Endung „.jnlp“) bereit
  • der Webserver / oder Ihr HELIUM V-Server, auf dem die Java-Anwendung bereitgestellt wird, muss für den Web-Start konfiguriert sein. Bitte wenden Sie sich an Ihren HELIUM V Betreuuer.
  • auf Ihrem Rechner ist das Java Runtime Environment installiert

Zur Nutzung des HELIUM V Web-Start stehen Ihnen zwei Möglichkeiten zur Verfügung:

 

 

1.)  Start über einen Browser

Geben Sie die URL http://HELIUMV_SERVER:8080/helium-client/ in einem Browser ein.
Auf Windows-Rechnern legen Sie mit Rechtsklick auf den Desktop, Neu, Verknüpfung eine neue Verknüpfung mit obiger Adresse an. HELIUMV_SERVER ersetzen Sie durch den Namen Ihres HELIUM V Servers.

 

Gegebenenfalls erscheint nun eine Meldung ähnlich der unten stehenden:

Grundsätzlich sollten Sie Ihr Java aktualisieren. Es kann jedoch die auf Ihrem Rechner installierte Javaversion für andere Programme zwingend erforderlich sein. Informieren Sie sich bitte bei Ihrem Systemadministrator.
Wenn nicht aktualisiert werden sollte, klicken Sie auf Später.
Ist Ihre Javaversion aktuell, so erscheint diese Meldung nicht.

 

Nun erscheint gegebenenfalls:

 

Bestätgen Sie diesen Dialog mit "Öffnen mit" - um die Abfrage in Zukunft zu vermeiden, können Sie den Haken bei "Für Dateien dieses Typs immer diese Aktion ausführen" setzen. 

Das Programm für die Ausführung des JNLP Files ist Teil des Java JRE-Pakets und wird automatisch ausgewählt.

Hinweis: Falls Java(TM) Web Start Launcher nicht vorgeschlagen wird, überprüfen Sie bitte die Java Installation Ihres Rechners. Besonders bei Ubuntu-Usern kann dies vorkommen, wenn OpenJDK installiert ist. Achten Sie bei der Installation darauf, dass das IcedTea-Plugin ebenso installiert wird.

 

Nun erscheint die Meldung:

 

Haken Sie beide Punkte an. Damit akzeptieren Sie, dass die von uns gelieferten Programme auf Ihrem Rechner installiert werden dürfen. Sollten Sie die Option nicht sehen, so haben Sie für diese Entscheidung zu wenig Rechte auf Ihrem Rechner. Bitten Sie Ihren Systemadministrator diese für Sie einzustellen.
Klicken Sie anschließend auf Ausführen.

 

 

Nach der Bestätigung startet Java und die Anwendung wird heruntergeladen und verifiziert. Hier erfolgt die Prüfung ob es Änderungen gegeben hat, sollte ein Update verfügbar sein, so wird die nun aktuelle Version geladen.

Im Anschluss daran startet der HELIUM V Client und Sie können sich wie gewohnt anmelden.

 

 

2.) Start über die Kommandozeile

 

Starten Sie die Kommandozeile und geben javaws http://HELIUMV_SERVER:8080/helium-client ein

Nun startet der Download und im weiteren Verlauf die Anwendung wie oben beschrieben.

 

Q: Welches Programm soll zum Öffnen der .jnlp-Datei verwendet werden?

A: Im Normalfall wird Java(TM) Web Start Launcher automatisch vorgeschlagen. Sollte diese Anwendung nicht zur Verfügung stehen, so prüfen Sie bitte Ihre Java-Installation.

Ubuntu: Bei der Installation von Java in Form von Open JDK muss das IcedTea-Plugin auch installiert werden!
