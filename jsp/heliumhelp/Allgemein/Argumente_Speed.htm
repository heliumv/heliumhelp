Argumente Speed

Gerade Logistik Pur Umsteiger sind im ersten Eindruck von der Geschwindigkeit von Helium V enttäuscht.

Von den bisherigen Installationen kann jedoch berichtet werden, dass insgesamt Helium V, bei richtiger Dimensionierung der Server und ausreichender Ausstattung der Clients, schneller ist, die Daten / Auswertungen deutlich früher zur Verfügung stehen als mit Logistik Pur.

 

Helium V ist von vorne herein Mehr-Mandatenfähig ausgelegt. D.h. die Zugriffe auf die Mandanten werden ausschließlich über die Berechtigungen gesteuert. Eine Abhängigkeit vom Betriebssystem ist nicht gegeben.

 

Mehrsprachige Benutzerschnittstelle. D.h. auch die angezeigten Feldnamen von Helium V können auf die bei Ihnen verwendeten Begriffe updatefähig angepasst werden.

 

Logistik Pur hat eine Zeichenbasierte Oberfläche. Helium V ist eine mit Swing gestaltete und ist eine sich in der Darstellung der Datenmenge automatisch anpassende modernste grafische Benutzer Oberfläche.

 

Sicherheitsaspekt. D.h. der Zugriff auf die Helium V Datenbank ist nur geschulten Anwendern (Insidern) möglich. Generell erfolgt der Zugriff über das Helium V Benutzer Rollen und Rechte System.

 

Helium V ist in Dreischichttechnik programmiert. Gegenüber der Einschichttechnik von Logistik Pur ist dies eine gänzlich andere Technik. Es bringt die Dreischichttechnik bei der ersten Betrachtung etwas Geschwindigkeitseinbusen, diese wird jedoch durch eine Reihe von Vorteilen, z.B. Skalierbarkeit, Stabilität aufgewogen. Bitte beachten Sie, dass sehr viele der handelsüblichen Programme nur in Zweischicht-Technik programmiert sind und damit ebenfalls die Rechnleistung auf den Clients liegt.

 

Betriebssystem unabhängig sowohl Client als auch Server z.B. Linux, MAC OS X, Windows

 

Dank Applikationserver ist Helium V Multi Client Fähig z.B. HTML Client, Web-Services usw.

 

Transaktions orientierte Datenzugriffe.

 

Pessimistik Transaktion Locking zu Ihrer Sicherheit.

 

Clientabsturz / Verbindungsabbrüche sind völlig gefahrlos, dank Datenbank, Applikationsserver und Transaktiontracking.

 

Ausdrucke rasch verfügbar.

 

Grafischer Editor für Reports.

 

Optimale Resourcenausnutzung, dank Lazy Loading. Dies bedingt jedoch, dass nach dem ersten Server Start deutlich erhöhte Ladezeiten für noch nicht benötigte Klassen, z.B. Auswahllisten, Druckvorschau auftreten.

Es kann bei manchen Funktionen vorkommen, dass beim Ersten Aufruf Daten initialisiert werden müssen, damit die danach folgenden entsprechend schnell funktionieren.

 

Direktfilter immer mit Enter auslösen, warum? Weil durch das Enter ein SQL Select ausgelöst wird. Würde bei jedem Tastendruck positioniert werden, würde dies einen deutlich höheren Netzwerkdatenverkehr / eine deutlich höhere Netzwerklast erzeugen.

Bitte beachten:

  • Wenn Sie Ihren Helium V Client auf einem Notebook betreiben, so schließen Sie das Notebook bitte an das Stromnetz an. Die heutigen Notebooks sind so gebaut, dass aus Gründer der Akku Laufzeit die Prozessorleistung bei Batterie Betrieb drastisch reduziert wird. Daraus ergibt sich eine drastische Geschwindigkeitsverbesserung, wenn Sie das Notebook am Stromnetz betreiben.

  • Auf manchen Windows2000++ PCs ist der Dienst ATI HotKey Poller installiert. Bitte deaktivieren Sie diesen Dienst. Er wird zu 99% nicht benötigt und verbraucht, insbesondere bei Clients wie Helium V unnötig viele Resourcen

  • Auf manchen Windows PCs läuft ein MS EasyKeyListener. Bitte auch dieses Programm / diesen Dienst deaktivieren
